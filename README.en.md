## Project Description

The EasySave project is part of the backup software development program for ProSoft. The goal is to design and deploy an efficient and maintainable backup software in several major and minor versions.

## Project Objectives

- Development of the EasySave backup software.
- Management of major and minor software versions (version 1.0, 1.1, 2.0, etc.).
- Provision of documentation for users and customer support.
- Ensuring code maintainability for future teams.

## Deliverables

### Deliverable 1 (EasySave version 1.0) - Date: 02/06/2025
- EasySave software version 1.0.
- One-page user documentation.
- Customer support documentation.
- UML diagrams for version 1.0.

### Deliverable 2 (EasySave versions 2.0 and 1.1) - Date: 02/14/2025
- Version 2.0 and version 1.1.
- UML diagrams for version 2.0.
- Updated documentation.

### Deliverable 3 (EasySave version 3.0) - Date: 02/28/2025
- Version 3.0.
- UML diagrams for version 3.0.
- Final documentation and Release Notes.

## Tools and Methods

- **Development Environment**: Visual Studio 2022 or higher.
- **Version Control**: GitHub for versioning and change tracking.
- **UML Modeling Tool**: ArgoUML for UML diagrams.
- **Programming Language**: C# with .NET 8.0 library.

## Development Conventions

- Adherence to naming conventions and development best practices.
- Readable, maintainable, and well-documented code.
- User documentation and technical support documentation.
- Minimization of duplicate code lines.
- Polished and intuitive user interface (UI).
